     │ HeRCULES Dataset Integration Plan                                                                                                          │                          
     │                                                                                                                                            │                          
     │ Context                                                                                                                                    │                          
     │                                                                                                                                            │                          
     │ The current Fusion Localization pipeline is built for RobotCar (6 sensors: 2 LiDAR + 3 Camera + 1 Radar-2D). HeRCULES has 3 sensors: 1     │                          
     │ LiDAR (Aeva), 1 Camera (stereo_left), 1 Radar (Continental 3D point cloud). The goal is to make the training/testing pipeline work with    │                          
     │ HeRCULES while maintaining multi-modal fusion capability.                                                                                  │                          
     │                                                                                                                                            │                          
     │ Key differences:                                                                                                                           │                          
     │ - Sensor count: 6 → 3 (only 1 of each modality)                                                                                            │                          
     │ - Radar is a 3D point cloud (Nx8), not a 2D image                                                                                          │                          
     │ - LiDAR is Aeva (8-channel .bin), xyz in first 3 columns                                                                                   │                          
     │ - Camera is 1440x1080 PNG (stereo_left only)                                                                                               │                          
     │ - Pose: [timestamp, x, y, z, qx, qy, qz, qw] quaternion format                                                                             │                          
     │ - Data locations: /data/drj/HeRCULES/{Library,Sports}/{*_01_Day,*_02_Night,*_03_Day}/                                                      │                          
     │ - Train/Val split: 01_Day + 02_Night = train, 03_Day = val                                                                                 │                          
     │                                                                                                                                            │                          
     │ Design Decisions                                                                                                                           │                          
     │                                                                                                                                            │                          
     │ 1. Radar as 3D Point Cloud → LiDAR Branch (Cylinder3D)                                                                                     │                          
     │                                                                                                                                            │                          
     │ Since HeRCULES radar is a 3D point cloud, process it through the same Cylinder3D sparse convolution pipeline as LiDAR. Both LiDAR and radar│                          
     │  produce [pt_features, voxel_coords, batch_size] 3-tuples, routed to Fusionmodel's LiDAR branch (len(input)==3).                           │                          
     │                                                                                                                                            │                          
     │ 2. 11-Tuple Format Preserved, 3 Forward Passes                                                                                             │                          
     │                                                                                                                                            │                          
     │ Keep the 11-tuple data format for compatibility. Missing sensors (right LiDAR, right camera, rear camera, 2D radar) are filled with zeros  │                          
     │ but never passed through the model. Training runs 3 forward passes instead of 6:                                                           │                          
     │ - Pass 1: LiDAR → model([pt_fea_lidar, vox_lidar, bs]) → loss1                                                                             │                          
     │ - Pass 2: Radar (as point cloud) → model([pt_fea_radar, vox_radar, bs]) → loss2                                                            │                          
     │ - Pass 3: Camera → model([stereo_left]) → loss3                                                                                            │                          
     │                                                                                                                                            │                          
     │ 3. Pose Conversion                                                                                                                         │                          
     │                                                                                                                                            │                          
     │ Use poses_to_matrices() from RALoc (util/pose_util.py) to convert quaternion → 4x4 matrix, then flatten to (N,12), then use                │                          
     │ tools/utils.py's process_poses() to convert to 6DoF [trans(3), log_quat(3)]. This ensures consistent pose normalization with RobotCar      │                          
     │ pipeline.                                                                                                                                  │                          
     │                                                                                                                                            │                          
     │ 4. No FusionModel Changes (Initial Version)                                                                                                │                          
     │                                                                                                                                            │                          
     │ Both LiDAR and radar point clouds share lidarmodaity bias. The Cylinder3D backbone handles both point cloud types. Separate bias terms can │                          
     │ be added later if experiments warrant it.                                                                                                  │                          
     │                                                                                                                                            │                          
     │ 5. Voxelization: fixed_volume_space=False                                                                                                  │                          
     │                                                                                                                                            │                          
     │ Radar has wider spatial extent (~120m) than LiDAR (~50m). Using data-adaptive bounds (fixed_volume_space=False) in pointclouddata() lets   │                          
     │ each point cloud auto-scale to [480, 360, 32] grid.                                                                                        │                          
     │                                                                                                                                            │

     │                                                                                                                                            │                          
     │ Context                                                                                                                                    │                          
     │                                                                                                                                            │                          
     │ The current Fusion Localization pipeline is built for RobotCar (6 sensors: 2 LiDAR + 3 Camera + 1 Radar-2D). HeRCULES has 3 sensors: 1     │                          
     │ LiDAR (Aeva), 1 Camera (stereo_left), 1 Radar (Continental 3D point cloud). The goal is to make the training/testing pipeline work with    │                          
     │ HeRCULES while maintaining multi-modal fusion capability.                                                                                  │                          
     │                                                                                                                                            │                          
     │ Key differences:                                                                                                                           │                          
     │ - Sensor count: 6 → 3 (only 1 of each modality)                                                                                            │                          
     │ - Radar is a 3D point cloud (Nx8), not a 2D image                                                                                          │                          
     │ - LiDAR is Aeva (8-channel .bin), xyz in first 3 columns                                                                                   │                          
     │ - Camera is 1440x1080 PNG (stereo_left only)                                                                                               │                          
     │ - Pose: [timestamp, x, y, z, qx, qy, qz, qw] quaternion format                                                                             │                          
     │ - Data locations: /data/drj/HeRCULES/{Library,Sports}/{*_01_Day,*_02_Night,*_03_Day}/                                                      │                          
     │ - Train/Val split: 01_Day + 02_Night = train, 03_Day = val                                                                                 │                          
     │                                                                                                                                            │                          
     │ Design Decisions                                                                                                                           │                          
     │                                                                                                                                            │                          
     │ 1. Radar as 3D Point Cloud → LiDAR Branch (Cylinder3D)                                                                                     │                          
     │                                                                                                                                            │                          
     │ Since HeRCULES radar is a 3D point cloud, process it through the same Cylinder3D sparse convolution pipeline as LiDAR. Both LiDAR and radar│                          
     │  produce [pt_features, voxel_coords, batch_size] 3-tuples, routed to Fusionmodel's LiDAR branch (len(input)==3).                           │                          
     │                                                                                                                                            │                          
     │ 2. 11-Tuple Format Preserved, 3 Forward Passes                                                                                             │                          
     │                                                                                                                                            │                          
     │ Keep the 11-tuple data format for compatibility. Missing sensors (right LiDAR, right camera, rear camera, 2D radar) are filled with zeros  │                          
     │ but never passed through the model. Training runs 3 forward passes instead of 6:                                                           │                          
     │ - Pass 1: LiDAR → model([pt_fea_lidar, vox_lidar, bs]) → loss1                                                                             │                          
     │ - Pass 2: Radar (as point cloud) → model([pt_fea_radar, vox_radar, bs]) → loss2                                                            │                          
     │ - Pass 3: Camera → model([stereo_left]) → loss3                                                                                            │                          
     │                                                                                                                                            │                          
     │ 3. Pose Conversion                                                                                                                         │                          
     │                                                                                                                                            │                          
     │ Use poses_to_matrices() from RALoc (util/pose_util.py) to convert quaternion → 4x4 matrix, then flatten to (N,12), then use                │                          
     │ tools/utils.py's process_poses() to convert to 6DoF [trans(3), log_quat(3)]. This ensures consistent pose normalization with RobotCar      │                          
     │ pipeline.                                                                                                                                  │                          
     │                                                                                                                                            │                          
     │ 4. No FusionModel Changes (Initial Version)                                                                                                │                          
     │                                                                                                                                            │                          
     │ Both LiDAR and radar point clouds share lidarmodaity bias. The Cylinder3D backbone handles both point cloud types. Separate bias terms can │                          
     │ be added later if experiments warrant it.                                                                                                  │                          
     │                                                                                                                                            │                          
     │ 5. Voxelization: fixed_volume_space=False                                                                                                  │                          
     │                                                                                                                                            │                          
     │ Radar has wider spatial extent (~120m) than LiDAR (~50m). Using data-adaptive bounds (fixed_volume_space=False) in pointclouddata() lets   │                          
     │ each point cloud auto-scale to [480, 360, 32] grid.                                                                                        │                          
     │                                                                                                                                            │

     │                                                                                                                                            │                          
     │ Class HerculesFusion(data.Dataset):                                                                                                        │                          
     │                                                                                                                                            │                          
     │ __init__(self, data_root, sequence_name, split, img_size=512):                                                                             │                          
     │ 1. Map sequence_name + split to sequences (Library: 01_Day+02_Night train, 03_Day val; Sports: Complex_*)                                  │                          
     │ 2. Load PR_GT/Aeva_gt.txt per sequence → poses_to_matrices() → flatten to (N,12)                                                           │                          
     │ 3. Build file lists for all 3 modalities:                                                                                                  │                          
     │   - LiDAR: LiDAR/np8Aeva/{timestamp}.bin                                                                                                   │                          
     │   - Camera: stereo_left/{timestamp}.png                                                                                                    │                          
     │   - Radar: Radar/multi_frame_w7/{timestamp}_multi_w7.bin                                                                                   │                          
     │ 4. KDTree match camera & radar timestamps to LiDAR timestamps (LiDAR is reference)                                                         │                          
     │ 5. Compute pose stats (mean_t, std_t) on train set, save/load {seq_name}_fusion_pose_stats.txt                                             │                          
     │ 6. Convert all poses via process_poses() → (N, 6) [trans(3), log_quat(3)]                                                                  │                          
     │ 7. Store Radar→LiDAR calibration matrix for coordinate transform                                                                           │                          
     │ 8. Compute LiDAR/Radar normalization stats (mean/std per xyz) on train set                                                                 │                          
     │                                                                                                                                            │                          
     │ __getitem__(self, index):                                                                                                                  │                          
     │ 1. Load LiDAR: np.fromfile(...).reshape(-1,8)[:,:3] → normalize → (N_l, 3)                                                                 │                          
     │ 2. Load Radar: np.fromfile(...).reshape(-1,8)[:,:3] → apply Radar→LiDAR calib → normalize → (N_r, 3)                                       │                          
     │ 3. Load Camera: PIL.open() → Resize(512) → CenterCrop(512) → ToTensor → Normalize → (3, 512, 512)                                          │                          
     │ 4. Get pose: self.poses[index] → (6,) float32                                                                                              │                          
     │ 5. Return 7-tuple: (camera, zeros(3,512,512), zeros(3,512,512), zeros(1,512,512), lidar_xyz, radar_xyz, pose)                              │                          
     │                                                                                                                                            │                          
     │ This 7-tuple maps to the cylinder_dataset.__getitem__() expectation:                                                                       │                          
     │ monoleft, monoright, monorear, radarimage, leftlidar, rightlidar, pose                                                                     │                          
     │ Where monoright, monorear are zero placeholders, radarimage (2D) is zero, and rightlidar is actually the radar 3D point cloud.             │                          
     │                                                                                                                                            │

     │                                                                                                                                            │                          
     │ pointclouddata(self, xyz): Copy from dataset_semantickitti.py lines 191-223. With fixed_volume_space=False, uses percentile-based bounds   │

     │ File 4: scripts/hercules_train.py — Training Script                                                                                        │                          
     │                                                                                                                                            │                          
     │ Purpose: Train FusionModel on HeRCULES with 3 forward passes per batch.                                                                    │                          
     │                                                                                                                                            │                          
     │ Structure (follows demo_train.py and TrainModel.py patterns):                                                                              │                          
     │                                                                                                                                            │                          
     │ # 1. Setup paths, set sys.argv for Combinedmodel.py argparse                                                                               │                          
     │ PROJECT_ROOT = ...                                                                                                                         │                          
     │ sys.argv = [sys.argv[0], '-y', CONFIG_PATH]                                                                                                │                          
     │                                                                                                                                            │                          
     │ # 2. Parse HeRCULES-specific args                                                                                                          │                          
     │ parser.add_argument('--data_root', default='/data/drj/HeRCULES/')                                                                          │                          
     │ parser.add_argument('--sequence', default='Library', choices=['Library', 'Sports'])                                                        │                          
     │ parser.add_argument('--batch_size', type=int, default=2)                                                                                   │                          
     │ parser.add_argument('--epochs', type=int, default=100)                                                                                     │                          
     │ parser.add_argument('--lr', type=float, default=0.0001)                                                                                    │                          
     │                                                                                                                                            │                          
     │ # 3. Build datasets directly (bypass data_builder.build())                                                                                 │                          
     │ train_dataset = HerculesFusion(data_root, sequence, split='train')                                                                         │                          
     │ train_cyl = hercules_cylinder_dataset(train_dataset, grid_size=[480,360,32])                                                               │                          
     │ train_loader = DataLoader(train_cyl, batch_size=bs, collate_fn=collate_fn_BEV, shuffle=True)                                               │                          
     │ # Same for val                                                                                                                             │                          
     │                                                                                                                                            │                          
     │ # 4. Build model                                                                                                                           │                          
     │ model = Fusionmodel().to(device)                                                                                                           │                          
     │                                                                                                                                            │                          
     │ # 5. Loss + optimizer (AtLocCriterion with learnable uncertainty)                                                                          │                          
     │ training_loss = AtLocCriterion(sax=0.0, saq=0.0, learn_beta=True)                                                                          │                          
     │ optimizer = Adam([model.parameters(), [training_loss.sax, training_loss.saq]], lr=lr)                                                      │                          
     │                                                                                                                                            │                          
     │ # 6. Training loop — 3 forward passes per batch                                                                                            │                          
     │ for epoch in range(epochs):                                                                                                                │                          
     │     for data in train_loader:                                                                                                              │                          
     │         # Extract data (11-tuple)                                                                                                          │                          
     │         actual_bs = data[10].shape[0]                                                                                                      │                          
     │         pt_fea_lidar = [torch.from_numpy(i).float().to(device) for i in data[2]]                                                           │                          
     │         vox_lidar = [torch.from_numpy(i).to(device) for i in data[1]]                                                                      │                          
     │         pt_fea_radar = [torch.from_numpy(i).float().to(device) for i in data[5]]                                                           │                          
     │         vox_radar = [torch.from_numpy(i).to(device) for i in data[4]]                                                                      │                          
     │         camera = torch.from_numpy(data[6]).float().to(device)                                                                              │                          
     │         labels = torch.from_numpy(data[10]).float()                                                                                        │                          
     │         transgt = labels[:, 0:3].to(device)  # following TrainModel.py convention                                                          │                          
     │         rotgt = labels[:, 3:6].to(device)                                                                                                  │                          
     │                                                                                                                                            │                          
     │         optimizer.zero_grad()                                                                                                              │                          
     │                                                                                                                                            │                          
     │         # Forward 1: LiDAR                                                                                                                 │                          
     │         trans1, rot1 = model([pt_fea_lidar, vox_lidar, actual_bs])                                                                         │                          
     │         loss1 = training_loss(trans1, rot1, rotgt, transgt)                                                                                │                          
     │                                                                                                                                            │                          
     │         # Forward 2: Radar (as point cloud, same interface)                                                                                │                          
     │         trans2, rot2 = model([pt_fea_radar, vox_radar, actual_bs])                                                                         │                          
     │         loss2 = training_loss(trans2, rot2, rotgt, transgt)                                                                                │                          
     │                                                                                                                                            │                          
     │         # Forward 3: Camera                                                                                                                │                          
     │         trans3, rot3 = model([camera])                                                                                                     │                          
     │         loss3 = training_loss(trans3, rot3, rotgt, transgt)                                                                                │                          
     │                                                                                                                                            │                          
     │         # Backward                                                                                                                         │                          
     │         loss = loss1 + loss2 + loss3                                                                                                       │                          
     │         loss.backward()                                                                                                                    │                          
     │         clip_grad_norm_(model.parameters(), max_norm=1.0)                                                                                  │                          
     │         optimizer.step()                                                                                                                   │                          
     │                                                                                                                                            │                          
     │     # Validation + checkpoint saving (same pattern as demo_train.py)                                                                       │                          
     │                                                                                                                                            │
